# 工具规格：edit (精细化编辑)

## 1. 功能目标
通过“寻找并替换”的方式对文件进行局部修改，避免全量重写大文件。旨在提高修改效率、节省 Token 消耗，并防止因模型截断导致的文件内容损坏。

## 2. 核心功能效果
*   **精确匹配替换**：要求代理提供一段在文件中**唯一存在**的旧字符串（`old_string`），并将其替换为新字符串（`new_string`）。
*   **安全性校验**：
    *   如果 `old_string` 在文件中不存在，工具报错并提示“未找到匹配项”。
    *   如果 `old_string` 在文件中存在多处且未指定 `allow_multiple`，工具报错并提示“匹配项不唯一”，要求代理提供更多上下文。
*   **原子性与一致性**：保持文件原有的缩进、换行符格式（CRLF/LF）。
*   **结果反馈**：执行成功后，返回修改位置前后的 Diff 摘要，供代理确认修改逻辑。

## 3. 参数定义 (JSON)
*   `file_path` (string): 文件的绝对路径。
*   `old_string` (string): 待替换的原始文本块（必须精确匹配）。
*   `new_string` (string): 替换后的目标文本。
*   `allow_multiple` (boolean, 可选): 是否允许替换所有匹配项，默认为 `false`。
*   `instruction` (string, 可选): 描述此次修改的意图，用于日志记录。
