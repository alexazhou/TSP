# 工具规格：write_file (全量写入)

## 1. 功能目标
创建新文件或完全覆盖现有文件的内容。它是初始化项目结构、生成配置文件或在确定需要全量更新小文件时的首选工具。

## 2. 核心功能效果
*   **自动目录创建**：如果目标文件所在的父目录不存在，工具应能够自动递归创建（类似于 `mkdir -p`），避免因路径不存在而报错。
*   **原子性写入**：采用“先写临时文件再重命名”的策略，确保在写入过程中即便程序崩溃，也不会留下损坏或部分写入的原始文件。
*   **安全保护**：
    *   **大文件警告**：如果代理尝试写入巨大的内容（如超过 100KB），工具应提示确认，建议其考虑分块处理。
    *   **写权限校验**：在执行前检查目标路径是否在允许的操作范围内（如受限的工作区目录）。
*   **文件属性维护**：默认创建的文件应具有合理的权限设置（如 `0644`）。

## 3. 参数定义 (JSON)
*   **`file_path`** (string): 文件的绝对路径。
*   **`content`** (string): 要写入的完整文本内容。
*   **`encoding`** (string, 可选): 文件编码格式，默认 `utf-8`。
